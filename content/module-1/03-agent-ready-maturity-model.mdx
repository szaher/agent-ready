## Overview

As we discussed in the previous lesson, agent readiness is not a binary state -- it is a spectrum. But how do you know where you stand on that spectrum? And more importantly, how do you chart a path forward? This lesson introduces the **Agent-Ready Maturity Model**, a five-level framework that helps you assess your current state, prioritize improvements, and track progress over time.

## Why a Maturity Model?

Every codebase is different. A startup's MVP has different needs than a decade-old enterprise monolith. A maturity model gives you:

- **A shared vocabulary** -- instead of debating whether your repo is "good enough" for agents, you can say "we are at Level 1 and targeting Level 2 this quarter"
- **Prioritized action** -- each level identifies the specific improvements that yield the highest impact at that stage
- **Measurable progress** -- you can track your movement along the model and celebrate wins

> The goal is not to reach Level 4 overnight. The goal is to know where you are and take the next meaningful step.

### Level 0: Unaware

At **Level 0**, the codebase has no preparation for AI agents. This is the default state of most repositories today.

**Characteristics:**

- No agent instruction file (no CLAUDE.md, .cursorrules, or AGENTS.md)
- Minimal or outdated README
- No documented conventions -- patterns exist only in team members' heads
- Tests may exist but are not reliably runnable with a single command
- Build steps require tribal knowledge ("ask Sarah how to set up the dev database")

**Agent experience at Level 0:** The agent spends most of its effort trying to understand the project. It guesses at conventions, frequently produces code that does not match team patterns, and cannot verify its own work. Developers spend more time correcting agent output than they would have spent writing the code themselves.

### Level 1: Foundational

**Level 1** represents the minimum viable preparation. The codebase has the basics in place for both humans and agents to get oriented.

**Characteristics:**

- A README with accurate setup instructions
- A working build command (`pnpm build`, `cargo build`, `go build`)
- A test suite that runs with a single command (`pnpm test`, `pytest`)
- Basic linting or formatting configured (ESLint, Prettier, ruff, gofmt)
- Some inline code comments explaining non-obvious decisions

**What to add to reach Level 1:**

- Write a README that includes: project purpose, setup steps, build command, test command
- Ensure `pnpm install && pnpm build && pnpm test` (or equivalent) works from a clean clone
- Add a linter if you do not have one

**Agent experience at Level 1:** The agent can build and test the project, which gives it a basic feedback loop. It still guesses at conventions, but at least it can tell when something is broken. Task completion rates improve significantly over Level 0.

### Level 2: Structured

**Level 2** is where agent-specific preparation begins. The codebase explicitly communicates its structure, conventions, and constraints to AI agents.

**Characteristics:**

- An **agent instruction file** (CLAUDE.md or equivalent) covering:
  - Project overview and architecture
  - Build, test, and lint commands
  - Code conventions and patterns
  - Files and areas that should not be modified
- Consistent directory structure with clear naming patterns
- Type checking enabled (TypeScript strict mode, mypy, Go's type system)
- Tests cover the critical paths, and new code is expected to include tests
- Pre-commit hooks or CI checks that enforce conventions

**What to add to reach Level 2:**

- Write a CLAUDE.md (we cover this in detail in Module 2)
- Document your top 10 coding conventions explicitly
- Enable strict type checking if your language supports it
- Add pre-commit hooks for linting and formatting

**Agent experience at Level 2:** The agent reads the instruction file and immediately understands the project context. It follows documented conventions, respects stated constraints, and uses the test suite to verify its work. Most straightforward tasks complete successfully on the first attempt.

### Level 3: Optimized

**Level 3** codebases have refined their agent integration based on real-world usage. They actively optimize for agent workflows and address common failure modes.

**Characteristics:**

- **Hierarchical instruction files** -- a root CLAUDE.md plus subdirectory-specific instruction files for complex areas
- **Exemplar patterns** -- documented reference implementations that the agent can copy from
- **Task-specific guidance** -- common workflows documented step-by-step ("to add a new API endpoint, follow these steps...")
- **Fast feedback loops** -- tests run in under 2 minutes, with unit tests completing in seconds
- **Error messages** that are actionable, not cryptic
- **Regular instruction file updates** -- the team reviews and refines agent instructions based on observed failures

**What to add to reach Level 3:**

- Add subdirectory CLAUDE.md files for complex modules
- Document the 5 most common development tasks as step-by-step workflows
- Identify and fix the top 3 things agents get wrong repeatedly
- Speed up your test suite

**Agent experience at Level 3:** The agent handles complex, multi-step tasks reliably. It knows where to find examples, follows established workflows, and rarely needs human correction for tasks within its documented scope. Developers trust the agent enough to review its output rather than rewrite it.

### Level 4: Autonomous

**Level 4** represents the frontier -- codebases where agents can handle entire features, from design through implementation and testing, with minimal human oversight.

**Characteristics:**

- **End-to-end task documentation** -- feature development workflows that agents can follow from spec to PR
- **Automated review gates** -- CI pipelines that validate not just correctness but convention adherence, performance, and security
- **Agent performance tracking** -- metrics on task completion rates, iteration counts, and human intervention frequency
- **Self-healing instructions** -- when agent failures are detected, the team updates instructions proactively
- **Integration with project management** -- agents can read issue descriptions and understand requirements directly

**Agent experience at Level 4:** The agent operates as a junior team member. Given a well-written issue, it can produce a complete implementation that passes all automated checks and requires only a final human review for architectural decisions and business logic correctness.

## Where to Start

If your codebase is at Level 0, do not try to jump to Level 3. Each level builds on the one before it. Here is a practical starting sequence:

1. **Today** -- ensure your build and test commands work reliably (Level 1)
2. **This week** -- write a basic CLAUDE.md with project overview and key commands (Level 2)
3. **This month** -- document your top conventions and add them to the instruction file (Level 2)
4. **This quarter** -- refine based on actual agent usage, add exemplar patterns (Level 3)

> Every improvement to your agent readiness is an improvement to your overall engineering practices. There is no wasted effort here.

## Assessing Your Own Codebase

Run through this quick checklist to estimate your current level:

- Can someone clone your repo and run tests with a single command? (Level 1)
- Do you have an agent instruction file? (Level 2)
- Are your coding conventions documented explicitly? (Level 2)
- Do you have step-by-step guides for common development tasks? (Level 3)
- Do you track how often agents succeed vs. fail on tasks? (Level 4)

The highest level where you can answer "yes" to all questions is approximately where your codebase sits. The first "no" tells you what to work on next.

## Summary

The **Agent-Ready Maturity Model** provides a five-level framework for assessing and improving your codebase's readiness for AI agents. From Level 0 (Unaware) to Level 4 (Autonomous), each level represents a meaningful improvement in how effectively agents can work in your repository. The model emphasizes progression over perfection -- even moving from Level 0 to Level 1 meaningfully improves how effectively agents work in your codebase.
